# v1.1.0 — Modulo de Servicios

## Resumen

Permitir a los propietarios configurar servicios (agua, luz, gas, internet, etc.) y distribuirlos entre sus propiedades de acuerdo a la cantidad de personas que viven en cada una. Cada mes se generan recibos editables por propiedad.

---

## Modelo de Datos

### Campo nuevo en `Property`

```typescript
// src/app/core/models/property.model.ts
interface Property {
  // ... campos existentes
  residentCount?: number; // cantidad de personas que viven en la propiedad (default 1)
}
```

### Nueva coleccion `services`

```typescript
// src/app/core/models/service.model.ts
interface Service {
  id?: string;
  ownerId: string;
  name: string;                     // "Agua", "Luz", "Gas", "Internet", etc.
  icon?: string;                    // material icon name (opcional)
  description?: string;
  isActive: boolean;                // si el servicio sigue vigente
  createdAt?: Timestamp;
  updatedAt?: Timestamp;
}
```

### Nueva coleccion `serviceAssignments`

Vincula un servicio a N propiedades. Cada assignment es una "configuracion" del servicio para un grupo de propiedades.

```typescript
// src/app/core/models/service-assignment.model.ts
interface ServiceAssignment {
  id?: string;
  ownerId: string;
  serviceId: string;
  serviceName: string;              // denormalizado para queries rapidas
  propertyIds: string[];            // propiedades que comparten este servicio
  distributionMethod: 'por_persona' | 'partes_iguales' | 'manual';
  createdAt?: Timestamp;
  updatedAt?: Timestamp;
}
```

### Nueva coleccion `serviceReceipts`

Recibo mensual generado por propiedad. Es el unico documento editable por el usuario.

```typescript
// src/app/core/models/service-receipt.model.ts
interface ServiceReceipt {
  id?: string;
  ownerId: string;
  serviceId: string;
  serviceName: string;              // denormalizado
  assignmentId: string;             // referencia al assignment
  propertyId: string;
  month: string;                    // 'YYYY-MM'
  totalAmount: number;              // monto total del servicio ese mes
  propertyAmount: number;           // monto calculado para esta propiedad
  residentCount: number;            // snapshot de personas al momento de generar
  isPaid: boolean;
  notes?: string;
  createdAt?: Timestamp;
  updatedAt?: Timestamp;
}
```

## Logica de Distribucion

Cuando el usuario genera recibos para un mes:

1. Toma el `totalAmount` ingresado para el servicio ese mes
2. Consulta `residentCount` de cada propiedad asignada
3. Calcula:

```
por_persona:
  total_personas = SUM(residentCount) de todas las propiedades del assignment
  propiedad_monto = totalAmount * (propiedad.residentCount / total_personas)

partes_iguales:
  propiedad_monto = totalAmount / cantidad_propiedades

manual:
  el usuario ingresa cada monto individualmente
```

4. Crea un `ServiceReceipt` por cada propiedad
5. Los recibos generados son editables (el usuario puede ajustar `propertyAmount`, `notes`, `isPaid`)

## Permisos de Colaborador

### Nuevo campo en `ColaboradorPermission`

```typescript
interface ColaboradorPermission {
  // ... campos existentes
  servicios?: boolean;   // undefined = true (backwards compat)
}
```

### Comportamiento

| Permiso | Descripcion |
|---------|-------------|
| `servicios !== false` | Puede ver servicios, assignments y recibos. Puede crear servicios, asignar a propiedades y generar/editar recibos |
| `servicios === false` | No ve la seccion de servicios en el sidebar ni puede acceder a la ruta |

## Rutas

```
/services                → ServiceListComponent (lista de servicios del owner)
/services/new            → ServiceFormComponent (crear servicio)
/services/:id            → ServiceDetailComponent (ver servicio + assignments + generar recibos)
/services/:id/edit       → ServiceFormComponent (editar servicio)
/services/:id/receipts   → ServiceReceiptsComponent (recibos del mes, editables)
```

Todas protegidas con `ownerGuard` (incluye colaboradores con permiso).

---

## Componentes

### 1. ServiceListComponent (`/services`)

- Lista de servicios del owner en cards
- Cada card muestra: nombre, icono, cantidad de propiedades asignadas, estado (activo/inactivo)
- Boton "Nuevo servicio" (solo si tiene permiso)
- Empty state cuando no hay servicios

### 2. ServiceFormComponent (`/services/new` y `/services/:id/edit`)

- Campos: nombre, icono (selector predefinido), descripcion
- Toggle activo/inactivo

### 3. ServiceDetailComponent (`/services/:id`)

- Info del servicio
- Seccion "Propiedades asignadas":
  - Selector multiple de propiedades (checkbox list)
  - Metodo de distribucion (radio: por persona / partes iguales / manual)
  - Boton guardar assignment
- Seccion "Generar recibos del mes":
  - Selector de mes (signal `selectedMonth`)
  - Input para monto total del servicio ese mes
  - Preview de distribucion (tabla con: propiedad, personas, monto calculado)
  - Boton "Generar recibos" → crea `ServiceReceipt` por cada propiedad
  - Si ya existen recibos para ese mes, muestra warning y opcion de regenerar

### 4. ServiceReceiptsComponent (`/services/:id/receipts`)

- Tabla de recibos del mes seleccionado
- Columnas: propiedad, personas, monto total, monto propiedad, pagado, notas
- Cada fila es editable inline:
  - `propertyAmount`: input numerico
  - `isPaid`: toggle
  - `notes`: input texto
- Auto-save on blur o boton guardar

## Flujo de Usuario

### Owner crea un servicio y genera recibos

1. Va a `/services` → clic "Nuevo servicio"
2. Llena nombre: "Agua", icono: `water_drop`
3. Guarda → redirige a `/services/:id`
4. En la seccion de propiedades, selecciona 3 apartamentos
5. Elige distribucion "Por persona"
6. Guarda assignment
7. En la seccion de recibos, selecciona mes "2026-03"
8. Ingresa monto total: $150.000
9. Ve preview:
   - Apto 101 (2 personas): $50.000
   - Apto 202 (3 personas): $75.000
   - Apto 303 (1 persona): $25.000
10. Clic "Generar recibos" → se crean 3 `ServiceReceipt`
11. Puede ir a recibos y editar montos individuales si necesita ajustar

### Owner edita un recibo

1. Va a `/services/:id/receipts`
2. Selecciona el mes
3. Edita el monto de una propiedad (override del calculo)
4. Marca como pagado
5. Agrega nota "Pago parcial, pendiente $10.000"

## Notas Tecnicas

- Los nombres de campo Firestore (`inmueblesUnidades`, `inmueblesPagos`, etc.) son legacy y no se renombran para evitar migraciones.
- El campo `servicios` en `ColaboradorPermission` sigue el patron: `undefined` = `true`.
- `ServiceReceipt.residentCount` es un snapshot del momento de generacion para mantener consistencia historica.
- `ServiceReceipt.totalAmount` tambien es snapshot para que al editar un recibo no se pierda el contexto.
- Usar `ServiceService` como nombre puede colisionar con el concepto de Angular service. Considerar `UtilityServiceService` o simplemente mantener `ServiceService` ya que el contexto es claro por el path de import.
